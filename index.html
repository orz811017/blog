心得-群暉二面

第三關 - 人資

這關算是人資跟你聊天認識你

上次面試有什麼感想？ 對公司的認知？ 為什麼要從電機系轉資工系？ 有沒有投其他公司？ 同學都投哪裡？

社團或家教之類的 拉賽一下 讓我心情放鬆

第三關

這關等很久XD 害我中途跑去上廁所 然後又被門把的靜電電了.... 又等了很久 考官才姍姍來遲

這考官看起來酷酷的 有抓髮蠟 問了問我的強項之後 沒多久就丟直球過來...

只問了問API跟Library的差異(不會QQ) 接著就換白板題了

  • 實做stack 要定義最基本的一些func variable是什麼
  • array滿了要怎麼擴充？可以加快嗎？
  • 要怎麼get max 或 min ？ 複雜度呢？

後面換了一題

  • 兩個sorted array 前一個size m+n 後面n個 是空的 另一個array長度則為n 要怎麼merge (in place)

一開始我從前面開始做.... 考官說好像不太直覺 有點小複雜 然後給了我hint說 那從後面開始會不會比較好 沒多久就寫好惹

接著就是我問問題的時間 問部門大概有哪幾個？產品只有NAS？（現在好像有Router） 然後午餐吃什麼..對方為什麼選群暉這樣.

他好像覺得我問題很少 就開始問我...興趣是什麼？ 我說打電動 之前有打過星海爭霸...看學長下圍棋或是下象棋 最近有再關注什麼新科技嗎？我說AR看起來很廢 比較有興趣的是阿法狗 跟 自動駕駛 有投哪幾間公司 為什麼想投之類的？我說半導體類的先等MTK之類的

臨走前問了static在c的功用是啥

第四關

等的時候一直很緊張 怕進來的是人資 結果進來的也是一個年輕人 主管級的 只能說群暉整體真的很年輕 每個進來都馬三十幾歲....

一開始先請我自我介紹 之後問了還記得mysql嗎？script呢？ 我說都忘光了... 問C/C++熟嗎 我說還可以 問一面試物件導向都問了些什麼 我回完之後

問了這些

  • template是在runTime還是complieTime決定的
  • java的generic 跟 template 是不是很像
  • dynamic 跟 static casting的差異
  • little endian & big endian 差在哪呢？有沒有例子可以說明
  • Verilog & C++ 特性差在哪裡
  • int,int*,void 參數長度是多少？x86跟ARM架構下這些參數長度有甚麼差?

以上我幾乎都不會崩潰 主管看我都不會都請我寫白板題惹

  • 實做 binary search
  • binary search 的array 改成二維的 請我要用pointer方式來做 不能透過[index]直接access (寫這題時候還因為前面被電太緊張 在上面亂寫一通XD 後來才改回來)
  • 有沒有辦法在加快呢（我上題寫兩層的 第二層binary search） 我問了一下資料是不是完全遞增的 就改成一層的

回答完之後就開始談offer 其實對Synology的薪水 我也沒有很懂 大概只有一個區間的概念 雖然落在我範圍的低標 不過我覺得就算惹 可能這關我回答的實在不是很好 而且有上就是種肯定了

問我對公司有什麼樣的問題 其實真的不太知道要問啥 要問的前面都問過了 就重新問了其中幾個 分紅之類的

問完之後就把我送出來惹

不過這關主管到說到了蠻中肯的話 做底層的 要有耐心的人格特資 如果之後分道這塊 可能是我要好好培養的一部份....

總結

薪水大概就MTK新人價 - 2XK 左右 因人而異 看ptt有人說最多可以到6開頭？ 分紅要第二年才領的到 加上去應該可以破百吧....XD

工時聽了一下還蠻正常的 七點多就會下班了 少數時間到九點十點

  • DS 大概 list stack queue tree是基本的
  • ALGO 考比較少 幾乎是沒有 題目大部分都動動腦就可以解完了
  • OS C++ ㄧ些基本知識倒是準備的挺充足的 最後一關被問倒 QQ

[心得]群暉

人生第一次面試RRR 其實蠻緊張的 出車站之後一直冒冷汗 下午一點半進去 進去也遇到一個新竹來的同學要來面試 不知道他結果如何 只能祝他好運惹

因為群暉在寸土寸金的台北車站附近 公司其實算蠻小的 櫃台也就小小的

後來被帶去小房間先填一下個人資料 學歷 預期薪資之類的 寫完之後約莫十分鐘面試官才進來

第一關

有兩位男生 大概都三十多歲

一開始請我自我介紹 講到project的時候就開始問問題了 主要是想知道

  • project負責的部分
  • project的大小(幾個人做)
  • 描述整個project的表達能力(?)

然後會找一個最代表性的project來問你 我是挑了研究所的計畫案

其實整個部分我覺得我沒有講得非常好 都是他們一直追問 才把整個project補充完整

有問的問題

  • multi-thread 好處? 為什麼你project要用multi-thread? multi-thread 的 program要注意甚麼
  • mutex & semaphore ? (忘記是不是在這邊問了)
  • thread & process ?為什麼thread context switch比較快?

問完之後就上白板題了

給你一個int dot(int x, int y) 的function 可以在二維平面上描整數點

請你implement一個畫圓的function drawcircuit (int cx , int cy ,int r)

一開始做了一個loop 從 cx-r開始出發 上下描點 透過畢氏定理 來算y的值

.問還可以再快一點嗎 我說可以在對稱一次 只做1/4圓

2.這樣是不是會有很多破洞 要怎麼補起來 我說按照離圓心的距離比例來做彌補

考官好像不是很滿意這個答案 就給了一個hint 從圓頂 cy+r 這邊可不可以想到甚麼 我大概知道意思 可是不會實作...QQ

3.sqrt比較慢 有沒有辦法不開根號就知道大約的整數值 有給些hint 我最後想一下 也不會 (後來考官出去之後我才想到很簡單QQ)

考官就說 那問題大概到這樣 問我有沒有甚麼問題

我問了一下最底層大概會做到哪 因為我覺得如果來群暉最比較上層的感覺會被電XD 應該會做到SoC之類的

最後我有說OSDI LAB有做kernel BOOT,Interrupt,MMU,Schedule,File system(IO)

這個好像蠻重要的 它們後來有多注記這一塊

第二關

兩關之間的休息時間蠻久的 害我很緊張 怕進來的是人資姊姊XD

後來進來的是一個年輕的工程師 我猜大概三十左右 一開始就問了問蠻多問題 大概印象是這些

  • 你這個project有用過那麼多語言(swift python c/c++ )?你覺得它們的差異好壞處在哪?
  • OO的好處及壞處?
  • 可以講一下OO的繼承跟多型嗎?
  • virtual function是?pure virtual function又是什麼?
  • 你的project有使用到OO嗎?為什麼?
  • array & list 的 優缺點?甚麼時候你會用array或list?
  • process中的 stack & heap 的差別?
  • 有聽過stack overflow 或是 bubble overflow?為什麼會有這問題?
  • 為什麼會產生segment fault?
  • TCP & UDP 的 差異?
  • 你有接觸過DataBase嗎?大概是怎麼做的?mysql之類的問題(早忘光了 直接說不會XD)

問完之後 考官笑著說 那我們來寫點題目吧XD

(1.) 兩個list在 某一點之後開始重疊到tail? 要怎麼判斷有沒有重疊? 我笑著說我早上才跟女朋友討論過這題XD 跟他說那要不要 要直接return分歧點嗎? 他想了想說好XD

先做了一個reverse的list 然後從tail一起反向出發找到next不一樣的分歧點 後來發現要複製題目list 不然題目做完一個reverse就被我破壞掉了 這樣時間複雜度(A+B) 空間也是(A+B)

接著問說有沒有比較省空間的 暴力法? 要我implement 並分析複雜度 做完之後 又問有沒有更好的方法

後來畫圖一下發現有更好的解法 用count計算兩條的長度 利用差值(落後領先的感覺)可以知道分歧點 這樣空間是O(1) 複雜度也只有(A+B)而已

(2.) 一個list 給你數字N 請找到 倒數第N個的node (跟上一題最後一個解法很像) 不過我看不太出來 後來考官 畫圖之後我才知道答案 (兩個ptr 一個指header 一個header+N)

(3.)最後一題是 n*n的矩陣翻轉90度 問要怎麼解 一開始額外開空間 蠻快就做完了 考官就問有沒有在in place的方法? 後來給了一個hint 說 跟第一關考的題目很像(旋轉的概念) 大概就知道解法了 只是我要記錄四個點 考官說可以更少 一個就夠了

然後問完之後問我有甚麼問題想問

我就不好意思的問了下班時間跟薪資XD (1.)就跟project一樣 deadline的時候大概10點多下班，平常平均大概七點~七點半下班 (2.)薪水要看部門，不太一樣

結束前多追問我平常會自己找題目來寫、會不會參加conference之類的？ 1.如果有人問或是在網路上不小心看到的話 就會想寫寫看 2.有時會想到一些idea就會想比賽 只是找不太到志同道合的人 (他有問會想一個人自己做嗎 我說不會 這樣project完整度不高 不夠大)

然後就請我稍等一下 大概約莫十來分鐘 同樣的考官就跟我說 今天面試先到這裡結束 離開前問我為什麼要轉資工 他說他也是電機轉資工的XD

然後回家的路上就蠻緊張的XD 怕沒電話打來 快六點的時候接到人資的電話 約下周五並在用104通知我一次(到現在沒收到QQ)
